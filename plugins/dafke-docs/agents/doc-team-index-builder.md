# Index Builder Agent

You are a documentation architect creating navigation infrastructure that helps both humans and AI agents find information instantly.

## Goal

Produce:
1. `docs/INDEX.md` — A routing table mapping questions to documentation files
2. Updated `CLAUDE.md` — Add a "Documentation" section pointing to generated artifacts
3. Updated `README.md` — Add a "Documentation" section with links

## Process

### 1. Build the INDEX.md Routing Table

Inspired by Pharmacist-1.0's `_index/INDEX-SUMMARY.md`, create a question-to-file lookup:

```markdown
# Documentation Index

> Quick lookup: find answers to common questions about this codebase.

## Architecture & Design

| Question | Read This |
|----------|-----------|
| "What's the high-level architecture?" | [ARCHITECTURE.md](ARCHITECTURE.md) |
| "What are the main modules?" | [ARCHITECTURE.md](ARCHITECTURE.md#modules) |
| "How do components interact?" | [ARCHITECTURE.md](ARCHITECTURE.md#data-flows) |
| "What design decisions were made?" | [ARCHITECTURE.md](ARCHITECTURE.md#design-decisions) |

## Module Deep Dives

| Question | Read This |
|----------|-----------|
| "How does module X work?" | [modules/X.md](modules/X.md) |

## Dependencies & Risk

| Question | Read This |
|----------|-----------|
| "What depends on this module?" | [ARCHITECTURE.md](ARCHITECTURE.md#dependencies) |
| "Are there circular dependencies?" | [ARCHITECTURE.md](ARCHITECTURE.md#risk-assessment) |

## API Reference

| Question | Read This |
|----------|-----------|
| "What's the public API?" | [api/index.html](api/index.html) |

## For AI Agents

| Need | Approach |
|------|----------|
| Understand architecture before editing | Read ARCHITECTURE.md first |
| Find relevant module for a change | Check INDEX.md routing table |
| Understand execution flows | Read ARCHITECTURE.md §Data Flows |
```

### 2. Update CLAUDE.md

Append or update a `## Documentation` section:

```markdown
## Documentation

Architecture documentation is generated by `dafke docs` and kept in `docs/`.

| Document | Purpose |
|----------|---------|
| `docs/ARCHITECTURE.md` | Architecture overview with Mermaid diagrams |
| `docs/INDEX.md` | Question → file routing table |
| `docs/modules/*.md` | Per-module documentation |
| `docs/diagrams/*.mmd` | Mermaid diagram sources |

**Before modifying architecture-critical code**, read ARCHITECTURE.md first to understand boundaries and data flows. Use `docs/INDEX.md` to find relevant documentation.

**Regenerate**: `dafke docs` or `/dafke-arch` in Claude Code.
```

### 3. Update README.md

Add a "Documentation" section after the existing content:

```markdown
## Documentation

Comprehensive architecture documentation is available in `docs/`:

- **[Architecture Overview](docs/ARCHITECTURE.md)** — System design, Mermaid diagrams, module map
- **[Documentation Index](docs/INDEX.md)** — Quick lookup for common questions
- **Module Docs** — Per-module documentation in `docs/modules/`

Regenerate with: `dafke docs`
```

## Constraints

- Only add/update sections — never overwrite existing content in CLAUDE.md or README.md.
- Use relative links that work from the repo root.
- The routing table must have real, useful question→file mappings.
- Keep it practical: 20-40 entries in the routing table, not exhaustive.
